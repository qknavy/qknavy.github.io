vuepress-theme-reco结合GitHub部署个人静态博客

Navy 2021-09-26 前端vue

作为一个开发人员，很多人习惯性整理自己在一段时间里的知识积累，即使不算正式的博文，作为手札随手记录平时的一些点点滴滴也是一种非常好的习惯。在真正用到的时候可以随时翻阅。所以，有人会选择买个个人空间或者买个云服务器来部署自己的博客。其实你知道吗，作为博客/文档主题的前端框架vuepress-theme-reco + github就可以方便地搭建自己的静态博客

# 1、vuepress-theme-reco项目的构建

# 1.1、环境安装

安装nodejs，可以采用安装包的方式安装，也可以采用二进制的方式安装，具体百度 (opens new window) 或者参考 nodejs官网 (opens new window)

# 1.2、脚手架安装

安装的方式有很多，比如：

  • npx安装
  • npm安装
  • yarn安装

这里以npm安装为例

#安装脚手架
npm npm install @vuepress-reco/theme-cli -g

#初始化我们的博客（下面my-blog是我们博客目录的名字，自己定义）
theme-cli init my-blog

# 2、博客配置

# 2.1、目录介绍

  • .vuepress/config.js

    这是vuepress的主要配置文件，包括插件、导航栏、等等，具体参考 官方文档 (opens new window) 或他人写的博客

    • https://www.cnblogs.com/glassysky/p/13387739.html
    • https://www.jianshu.com/p/b6508c21c230

  • .vuepress/public

    存放一些静态资源文件，比如logo之类的

  • blogs

    博客文章目录，博客类的文章都放在这个目录下，最终会在主页的文章列表中体现

  • docs

    文档目录，可以归档一些自己的笔记，1.6版本之前不会在主页的文章列表里体现出来，但是1.6之后blogs和docs目录下的文章都会出现在主页的列表里

  • node_modules

    nodejs的依赖模块

  • package.json

    nodejs的项目配置文件

  • README.md

    博客首页相关的自定义配置

# 2.2、几个主要的配置

  • 加载页文字

config.js > title|description

  • 首页配置

README.md > heroText | tagline

# 2.3、插件的安装

参考：https://vuepress-theme-reco.recoluan.com/views/plugins/

# 3、部署到GitHub

# 3.1、创建博客工程

在自己的GitHub仓库下创建一个工程（功能限制，得是public，所以为了安全建议将文档源文件和要部署的静态页面文件分开），用于存放博客文件

PS：

工程名字为[账号].github.io，比如我的GitHub账号是qknavy，所以我创建的仓库就是qknavy.github.io

# 3.2、添加部署脚本

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd ./public

git init
git remote add origin https://[ACCESS_TOKEN]@github.com/qknavy/qknavy.github.io.git #换成自己工程的路径，
#`ACCESS_TOKEN`令牌可以在https://github.com/settings/tokens 创建

git add -A
git commit -m 'deploy'

git push -f origin master	#push到自己指定的分支，我这里直接push到了master

cd -

# 3.3、添加GitHub Page

在自己的项目setting下，进入pages配置页，选择对应的分支和目录，保存即可发布。

发布完成可以通过https://[账号].github.io访问，也可以再下边Custom domain配置自己的域名，域名需要解析到GitHub的dns上，具体的ip之类的点及右边的Learn more查看

# 3.4、GitHub Action自动部署

经过上述操作，可以手动执行deploy.sh脚本手动部署，但是不符合程序员的的习惯，我们的终极目标是只负责写文档提交，剩余的工作希望自动完成比较好。

GitHub提供了自动部署的功能，结合Action可以让我们的博客在我们提交了文档之后自动编译、部署。

具体参考官方介绍：

github自动部署 (opens new window)
