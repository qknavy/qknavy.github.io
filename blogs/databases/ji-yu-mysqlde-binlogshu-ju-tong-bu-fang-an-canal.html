基于mysql的binlog数据同步方案-canal

Navy 2021-03-23 canaldatabase

本文简单介绍canal的基本原理和用法

在如今的系统设计的过程中，经常会用到各种数据存储技术，比如关系型数据库、redis或ES缓存方案。但是在我们的业务系统中，又经常需要考虑到数据的一致性问题，我们是究竟是采用CP还是AP主要还是根据我们的业务场景而定。在一般的读多写少的场景下，为了保证数据库和缓存数据的一致性，基于binlog的canal是一种比较好的解决方案。

# 5、k8s部署

# 5.1、canal-admin

canal-admin.yaml

apiVersion: apps/v1
kind: Deployment
metadata:
  annotations:
    deployment.kubernetes.io/revision: '1'
  generation: 1
  labels:
    app: r3-canal-admin
  name: canal-admin
  namespace: qknavy
spec:
  progressDeadlineSeconds: 600
  replicas: 1
  revisionHistoryLimit: 10
  selector:
    matchLabels:
      app: r3-canal-admin
  strategy:
    rollingUpdate:
      maxSurge: 25%
      maxUnavailable: 25%
    type: RollingUpdate
  template:
    metadata:
      labels:
        app: r3-canal-admin
    spec:
      containers:
        - env:
            - name: server.port
              value: '8089'
            - name: spring.datasource.address
              value: dbhost:dbport
            - name: spring.datasource.database
              value: qknavy_canal_admin
            - name: spring.datasource.username
              value: username
            - name: spring.datasource.password
              value: passwd
          image: 'canal/canal-admin:v1.1.4'
          imagePullPolicy: Always
          name: r3-canal-admin
          resources:
            requests:
              cpu: 250m
              memory: 512Mi
          terminationMessagePath: /dev/termination-log
          terminationMessagePolicy: File
      dnsPolicy: ClusterFirst
      restartPolicy: Always
      schedulerName: default-scheduler
      securityContext: {}
      terminationGracePeriodSeconds: 30
