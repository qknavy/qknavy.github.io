mybatis字段自动映射及数据的逻辑删除

Navy 2020-09-09 mybatis-plusdatabase

在执行数据库操作的时候，为了数据安全，我们有时候会预留一些字段记录数据的创建时间、最后修改时间以及是谁操作的，但又不想每次都手动去设置；另外有些数据因为某些原因（安全、业务需要）需要做逻辑删除

# 1、定义一个实体映射的模板

public abstract class AbstractEntity implements Serializable {

    @TableField(value = "created_by", fill = FieldFill.INSERT)
    private String createdBy;

    @TableField(value = "created_date", fill = FieldFill.INSERT)
    private long createdDate = System.currentTimeMillis();

    @TableField(value = "last_modified_by", fill = FieldFill.INSERT_UPDATE)
    private String lastModifiedBy;

    @TableField(value = "last_modified_date", fill = FieldFill.INSERT_UPDATE)
    private long lastModifiedDate = System.currentTimeMillis();

    @TableField(value = "del_flag")
    @TableLogic(value = "0", delval = "1")
    private int delFlag;

    //省去了getter、setter方法
}

# 2、定义用于自动处理的 MetaObjectHandler

@Component
public class MetadataHandler implements MetaObjectHandler {
    /*
        对应数据库字段：
        created_by
        created_date
        last_modified_by,
        last_modified_date
     */
    @Override
    public void insertFill(MetaObject metaObject) {
        Optional<String> currentUserUid = SecurityUtils.getCurrentUserUid();
        String loginName = null;
        if (currentUserUid.isPresent()) {
            loginName = currentUserUid.get();
        } else {
            loginName = UserConstants.SYSTEM_UID;
        }
        long now = System.currentTimeMillis();
        this.setFieldValByName("createdDate", now, metaObject);
        this.setFieldValByName("createdBy", loginName, metaObject);
        this.setFieldValByName("lastModifiedDate", now, metaObject);
        this.setFieldValByName("lastModifiedBy", loginName, metaObject);
    }

    @Override
    public void updateFill(MetaObject metaObject) {
        Optional<String> currentUserUid = SecurityUtils.getCurrentUserUid();
        String loginName = null;
        if (currentUserUid.isPresent()) {
            loginName = currentUserUid.get();
        } else {
            loginName = UserConstants.SYSTEM_UID;
        }
        long now = System.currentTimeMillis();
        this.setFieldValByName("lastModifiedDate", now, metaObject);
        this.setFieldValByName("lastModifiedBy", loginName, metaObject);
    }
}

# 3、逻辑删除配置

想要这个自动处理生效，还需要注入ISqlInjector这个bean

//省去其它代码
@Bean
public ISqlInjector sqlInjector() {
    return new LogicSqlInjector();
}

# 4、逻辑删除注意事项

逻辑删除的时候只能调用mybatis自己的删除方法，而不能调用自己在mapper定义的接口方法，否则逻辑删除不会生效。因为mybatis是通过ISqlInjector这个接口的动态代理去完成了，而我们自己定义的方法意味着直接执行sql语句
