# 调用

在dubbo接口调用的时候我们通过@Reference注解（注意是dubbo的注解，不是jdk的注解，千万别引错了）。

此注解有一个配置validation,如果需要开启校验的话需要设置成：validation="true"

通过以上步骤，我们的dubbo接口就具备了自动参数校验的规则，剩下的就是自己对校验不通过的结果进行相应的处理

# 4、自定义自己的校验规则

# 自定义注解

@Target({ElementType.FIELD,ElementType.PARAMETER})  //这里作用于属性和参数
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Constraint(validatedBy = {IsPhoneConstraintValidator.class})   //真正的校验器逻辑类
public @interface IsPhone {
    String message() default "手机号格式不正确";

    Class<?>[] groups() default {};

    Class<? extends Payload>[] payload() default {};//对于实体对象作用的
}

# 自定义自己的校验器规则

/**
 * 验证是否是手机号
 */
public class IsPhoneConstraintValidator implements ConstraintValidator<IsPhone, String> {
    //简单验证手机号
    public Pattern pattern = Pattern.compile("^1[3456789]\\d{9}$");

    @Override
    public boolean isValid(String value, ConstraintValidatorContext constraintValidatorContext) {
        //真正的校验逻辑
        if (value == null || value.length() == 0) {
            return true;
        }
        return pattern.matcher(value).matches();
    }
}
