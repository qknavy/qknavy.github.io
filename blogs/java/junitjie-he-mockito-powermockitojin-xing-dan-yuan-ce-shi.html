Junit结合Mockito、PowerMockito进行单元测试

Navy 2021-02-26 test单元测试

本文只是简单介绍一下Junit结合Mockito、PowerMockito进行单元测试

# 1、初衷

在开发的过程中，为了保证代码的正确性，我们开发人员往往需要对自己写的代码进行单元测试。常见的方式就是Junit，但是如果只是直接使用Junit，还是会存在一些问题：

  • 基于容器，单元测试启动慢

比如我们一般都是基于springboot开发，而如果直接使用spring-boot-starter-test进行单元测试的使用，我们经常是在测试类上打两个注解@RunWith(SpringRunner.class)、@SpringBootTest(Bootstrap.class)。

但是这个时候相当于在测试的使用需要先启动我们的spring容器，特别是如果需要加载的东西比较多的时候就更慢了

  • 数据隔离

在单元测试的时候，不管测试执行的结果是成功还是失败，我们的数据不能影响当前环境的正常数据，如果简单实用Junit的话，在@Before需要准备数据，@After需要清理数据

  • 依赖比较重

在我们的业务代码逻辑中，我们经常是不同服务之间、不同层之间、方法之间的相互调用，甚至还会涉及到一些外部资源，比如：数据库、MQ、缓存。

所以，有时候我们的调用链路是非常长的，这时候想要去覆盖就比较麻烦。但是单元测试的目的就是为了保证我们代码的质量，如果我们每个类、每个方法都能保证在各种场景下逻辑完全正确，那么我们的代码质量就可以得到保障了。

Mockito就是一个很好的选择

# 2、Mockito的特性

Mockito最好的特性就是我们可以mock各种数据，从而不对当前环境的数据造成任何影响，而且启动执行也很快（个人看法）

# 3、基本用法

引入spring-boot-starter-test就默认引入了Mockito相关的类（不包括PowerMockito）

# 3.1、测试public方法

  • 定义的待测试的接口和实现类

     * 获取测试数据
     * @param userName

     * @return
     */

}


@Service
public class TestServiceImpl implements ITestService {

    @Override
    public String getTestData(String userName, String password) {

    @RunWith(PowerMockRunner.class)
@PrepareForTest({
        PrivateServiceImpl.class
})
public class PrivateServiceImplTest {

    @InjectMocks
    private PrivateServiceImpl privateService;

    @Test
    public void testShowPrivateName(){
        Method spy = PowerMockito.method(PrivateServiceImpl.class, "showPrivateNames", String.class);

        try {
            Object qknavy = spy.invoke(privateService, "Qknavy");
            assert "hello, private Qknavy".equals(qknavy);
        } catch (IllegalAccessException e) {
            e.printStackTrace();
        } catch (InvocationTargetException e) {
            e.printStackTrace();
        }
    }
}

    @RunWith(PowerMockRunner.class)
@PrepareForTest({
        PrivateServiceImpl.class
})
public class PrivateServiceImplTest {

    @InjectMocks
    private PrivateServiceImpl privateService;

    @Test
    public void testCallStatic(){
        PrivateServiceImpl spy = PowerMockito.spy(this.privateService);
        try {
            //生命mock某个类的某个static类型方法，参数类型是什么
            PowerMockito.mockStatic(PrivateServiceImpl.class, String.class);
						//下面表示任意字符串作为参数调用PrivateServiceImpl.showStaticNames方法都返回“小刘”
            PowerMockito.when(PrivateServiceImpl.showStaticNames(Mockito.anyString())).thenReturn("小刘");
						//执行具体的待测试的方法
            String result = spy.callStatic("liuyuanskun");
						assert "calling 小刘".equals(result);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

# 3.5、对public类型的子方法进行mock

@Service

    }

    @RunWith(PowerMockRunner.class)
@PrepareForTest({
        PrivateServiceImpl.class
})
public class PrivateServiceImplTest {

    @InjectMocks
    private PrivateServiceImpl privateService;

    @Test
    public void testCallPrivate(){
        //获得一个代理对象
        PrivateServiceImpl spy = PowerMockito.spy(this.privateService);
        try {
          	//当执行代理对象的对应私有方法的时候，不同的参数对应不同的输出
            PowerMockito.when(spy, "showPrivateNames", "liuyuankun").thenReturn("刘元坤");
            PowerMockito.when(spy, "showPrivateNames", "qknavy").thenReturn("乾坤纳威");
						//执行被测试的方法
            String result = spy.callPrivate("liuyuankun");
            assert "calling 刘元坤".equals(result);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}

# 3.7、对于多层调用（比如Service调dao）

经常在我们的业务逻辑中，会不同层或者不同服务之间的调用，包括不同类之间的方法调用。如果我们只关注当前这个方法的逻辑，则可以对其他资源进行mock，比如：

  • 准备待测试的类

    @Service
public class TestServiceImpl implements ITestService {

  	@Autowire
    private TestDataDao testDataDao;

    @Override
    public String getTestData(String userName, String password) {
        return testDataDao.getData(userName, password);
    }
}


@Component
public class TestDataDao {
    public String getData(String userName, String password) {
        return "fail";
    }
}
