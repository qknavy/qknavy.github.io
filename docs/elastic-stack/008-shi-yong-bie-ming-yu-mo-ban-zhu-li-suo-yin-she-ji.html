8、使用别名与模板助力索引设计

Navy 2020-09-03

ElasticSearch是文档型数据库，不是关系型数据库，不具备事务特性，所以千万不要运用在严格事务性场景的应用项目中

# 1、索引别名深入解读

# 1.1、索引别名（alias）是什么？

# 概念介绍

ES索引创建之后，本质上都是同步别名访问的，默认情况下索引别名等同于索引名称（创建索引的时候相当于会同步创建一个同名的索引别名）

@flowstart

a=>operation: Client应用端

b=>operation: Proxy代理

c=>operation: Data数据

a(right)->b

b(right)->c

@flowend

# 应用场景

索引别名本质上就是索引的代理

  • 业务新旧索引切换

    比如老的索引结构变化，只需要将别名指向新的索引。在业务场景中需要提前创建好别名（也可以动态的方式去创建）

  • 索引访问资源的隔离

    索引的别名可以指向很多个索引，别名和索引是多对多的关系，可以通过别名的方式去做到灵活切换。（比如读写分离）

  • rollover滚动创建索引基于别名实现

# 1.2、别名绑定

# 别名创建方式

索引创建的时候默认会创建并绑定一个和自己名字一样的别名。但是有时候需要我们手动去创建和维护。

# 别名绑定特性

别名和索引是多对多的关系，一个索引可以绑定多个索引别名，同时一个索引别名也可以绑定到多个索引。

# 使用

除了上面这种方式之外，我们还可以手动绑定一个索引，比如：

PUT qknavy-company-0022/_alias/qknavy_c
1

这条指令就会给上面创建的索引qknavy-company-0022创建并绑定一个别名。

GET qknavy_c/_search

# 1.6、别名高级使用

# 1.6.1、别名查询过滤

比如在电商场景，所有的商品都在一个索引里边，但是我们很多部门只关心自己所负责的品类，比如衣服类、鞋子类、食品类等

这个时候就可以通过别名查询过滤返回不同的品类给对应的用户，给人的感觉就是数据存在不同的索引中一样。

换句话说，这也是多个索引别名对应同一个索引的场景

# 2、索引模板深度使用

# 2.1、索引模板是什么？

# 2.1.1、定义

创建索引设置索引字段的行为，需要设计索引分片与副本，以及其他很多高级的设置。每次创建都需要重复的工作，不是很方便。索引模板就是一种规则，设定好之后，无论之后任何时候创建索引都一句模板设置执行，是一种自动化的机制。

# 2.1.2、应用领域

  • 规范索引行为

  • 提高开发效率

  • ES很多高级功能基于此，比如ILM索引自动化管理

# 2.1.3、模板基本使用

创建一个模板需要制定对应的索引匹配规则以及索引的映射、设置等...

# 2.1.4、模板的内容

模板可以覆盖更新，要注意模板冲突

# 匹配规则：

  • 根据索引名称匹配（前缀/后缀）、正则，支持多个

  • 索引模板的执行顺序

    在创建模板的时候可以设置order属性控制模板的优先级。数值越大，优先级越高

  • 多索引模板混合匹配

    最终的字段映射为多个匹配到的模板的字段组合

# 索引模板的索引设置：

索引模板等同于索引本身，可以提前做一些设置，如分片、副本数、刷新时间等等

# 模板字段的映射：

索引m不安的第二个目的就是设置索引字段，控制字段的映射行为

# 模板别名绑定：

索引别名是一个非常便利的特性，有很多的索引可以提前在索引模板设置里面绑定，之后创建的索引自动绑定别名
