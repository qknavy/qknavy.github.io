# 元数据字段有什么用？

概念：一条文档数据之外的数据叫元数据，元数据是属于es的产物，不属于Lucene。

  • _type

    索引类型，跟随在索引名称之后，7.0之前必填，7.0之后固定值_doc，8.0之后移除

  • _id

    • 文档数据id，一条文档数据必须有一条id
    • 一个索引中必须是唯一的。
    • 默认采用自动生成，规则参考objectid规则
    • 可以自己指定，不限制类型，可同时存储数字与字符。ID长度限制512字符
    • ES自带的id比我们指定的业务id性能要强

  • _source原数据

    索引原数据元数据，属性控制存储原始数据

  • _route路由

# 3.2、动态字段模板（dynamic template）

字段检测动态模板，通过设置一定的匹配规则，固定下字段的类型与属性。（回顾以前，我们都是自己在mapping里边通过properties设置每个字段的类型和属性，但是如果字段特别多，大多数类型都是可知，而自动检测又会浪费一定的空间，这时候可以通过动态字段模板来做）

  • 字段类型匹配机制

    定义一种规则，一句json字段类型来匹配设定字段类型与属性。

    PUT qknavy-company-0018

  "mappings": {
    "dynamic_templates": [

        "integer_type": {
          "match_mapping_type": "long",
          "mapping": {
            "type": "integer"
          }
        }
      },
      {
        "string_type": {
          "match_mapping_type": "string",
          "mapping": {
            "type": "keyword",
            "ignore_above": 255

      }
