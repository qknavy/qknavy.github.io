10、高效的更新数据实战

Navy 2020-09-06

ElasticSearch是文档型数据库，不是关系型数据库，不具备事务特性，所以千万不要运用在严格事务性场景的应用项目中

# 1、数据的更新过程与内部机制

# 1.1、数据更新的路由过程

es根据路由计算出分片，路由可指定先是主分片，然后才是副本分片，具体过程参考 数据的写入

# 1.2、数据更新的本质

数据更新的本质操作：

  • 查询数据

  • 标记删除

    首先将原来的数据标记为删除状态，并记录版本号

    覆盖更新：根据更新的内容直接插入一条新的数据

    局部更新：和查询的内容进行merge合并，然后再写入

  • 插入数据

删除操作实际上是三个步骤，但是这三个步骤是可以保证”事务性“的

# 2、常用数据更新方式

# 2.1、单条覆盖更新

  • PUT方式单条数据覆盖式更新，由客户端完成所有数据的组装，服务端认可数据的完整性，然后执行覆盖
  • POST方式更新，等同于PUT

局部更新的时候更新数据的时候不需要提交完整的字段数据。其实本质上也是覆盖式更新，只是由服务端完成数据的合并。

局部更新支持upsert和script

# 2.4、数据更新的参数解读

  • REQ请求参数

    • refresh：是否立即刷新索引
    • retry_on_conflict，更新冲突充实次数，默认为0

  • 响应参数

    • result：操作结果（Created（创建成功）、Updated（修改成功）、Deleted（删除成功）、noop（没有变动））
    • version：数据的版本号
    • seq_no：数据更新序号
