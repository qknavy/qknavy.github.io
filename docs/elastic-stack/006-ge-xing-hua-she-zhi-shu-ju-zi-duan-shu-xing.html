# 概念介绍：

官网介绍 (opens new window)

# 1.2、是否存储（store）

这个属性是lucene里的概念

_source是ES的概念

# 概念介绍：

官网介绍 (opens new window)

ES默认原始数据存储在_source里。如果需要检索的字段比较少的时候也可以在Lucene里边存储一份，在检索的时候可以提高效率（因为检索的时候会先从Lucene里边通过倒排索引找到id，找到id之后根据_source里边的id找到原始数据）。通过这个关键字去设置。

取值范围：true/false，默认值为false

# 应用场景：

原始数据_source被禁用，但是想保留一份原始数据，要检索的字段比较少，想要提升检索效率的时候可以考虑这种方案

# 使用方式：

  • 创建mapping（禁用_source）

{
  "mappings": {
    "_source": {
      "enabled": false
    },
    "properties": {

        "type": "text",
        "index": true
      },
      "companyName": {
        "type": "text",
        "index": true
      }
    }

# 概念介绍：

官网介绍 (opens new window)

是否被检索，比较类似index，但是index作用在某个字段上，比较精细化，enabled主要针对的是object类型。

取值范围：true/false，默认为true

# 应用场景：

# 1.4、能否聚合/排序（doc_value）

# 概念介绍：

官网介绍 (opens new window)

传统数据库为行式数据库，检索的时候需要一行一行地去查找。ES为列式数据存储，检索的时候只需要检索指定的列，所以数据加载量少很多，效率也会高很多。

ES存储的数据除了索引文件之外默认会保存两份数据：一份原始数据（_source）、一份列式存储(doc_value)。另外如果我们开启了store，它还会在Lucene里面保存一份

取值范围（true/false），默认取值true

# 应用场景：

  • 基于该字段做聚合分析

  • 基于该字段做排序

  • 由于是限制聚合或排序，所以一般是用在数值类型字段上

# 使用方式：

如果没有开放的话在使用的时候对这个字段进行了聚合操作或者排序的话会报错

# 1.5、空值默认（null_value）

官网介绍 (opens new window)

# 概念介绍：

空置默认值，ES容许可以不设置字段，没有数据的必须填充选项。但是它不会存储在_source里，而只会设置到索引里边用来做检索。

# 应用用途：

如果要对某个字段进行检索，但是不需要保存原始数据的时候可以用到。

设置默认值也可以避免程序出错。

# 1.7、动态控制（dynamic）

官网介绍 (opens new window)

# 概念定义：

ES在默认情况下mapping是动态扩展的，可以根据我们插入的数据的格式自动完成映射，如果需要手动控制的话就可以用到dynamic属性。

取值范围：true/false/strict，默认值true

# 使用方式：

  • 应用于mapping全局控制

  • 应用于字段局部控制

  • 如果一个字段的dynamic属性设置为false，那么它下面再动态插入的子对象只能作为数据字段，不能做检索、聚合等操作，不会出现在mapping里边

  • 如果一个字段的dynamic属性设置为strict，那么它下面再有新的子对象或者属性的时候，数据是无法插入的，执行的时候会报错

# 1.8、字符串长度限制（ignore_above）

官网介绍 (opens new window)

# 概念介绍：

keyword类型下， 字符过长的时候检索意义不大，索引会被禁用，数据不可被检索。所以就需要对字符串的长度进行限制。（如果我们的字段设置的是text类型，并且没有做其它设置的时候，es就会给我们动态生成keyword类型的映射，并且长度控制到256个字节）

# 应用领域：

基于keyword类型检索应用，超过限定长度就无需检索（注意：不是截取，是直接检索失效）
