5、特殊数据字段类型应用实战

Navy 2020-08-27

ElasticSearch是文档型数据库，不是关系型数据库，不具备事务特性，所以千万不要运用在严格事务性场景的应用项目中

# 1、关联字段类型

# 1.1、父子类型（join）

# 概念定义：

数据存在父子关系：一条数据可以有多条子数据；一条子数据仅一条父数据

比如一个公司有多个子公司，一个父亲有多少个儿女，一个问题有多少个答案等等

# 优点：

数据更新分离影响。比如需要修改子对象，只需要修改子对象即可，如果使用数组或者对象的话需要修改整个文档（特别是文档字段非常多的时候开销特别大）

# 缺点：

不能动态映射，需要手动mapping

# 注意事项：

  • 父子数据必须在同一分片上
  • 父子层级深度要注意，不要太深
  • 一个索引仅容许一对父子关系约束
  • 一个父关系可以支持多个子对应关系

# 使用场景：

具备父子关联的数据需求

# 1.2、嵌套类型（Nested）

# 概念定义：

数组键值对查询会出现数据查询误差，nested的出现就是为了解决键值对查询误差

# 内置实现：

将嵌套的一条数据拆分成多条数据存储，并通过内置查询关联合并为一条数据。

# 缺点：

数据维护比较麻烦，因为es底层会拆分

# 注意事项：

单条嵌套数据量条数限制，不要嵌套太深，会影响性能。默认限制1024

# 使用场景：

一般情况不建议使用，除非确实有这个必要

# 使用：

# 2.1、别名类型（alias）

es中别名有两种类型：索引别名和字段别名

# 别名应用：

字段代理、字段指向，避免直接暴露原始字段

# 注意事项：

数据不可更新，别名不存储任何数据，仅仅是路径指向

别名可以有多个，不限制

# 使用场景：

有些字段不想对外暴露（比如命名很丑陋），需要将规范的字段返回给前端的时候。

有的索引的某个字段嵌套很深，操作的时候为了简写，可以通过别名方式来操作
